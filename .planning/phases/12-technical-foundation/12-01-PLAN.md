<task type="auto">
  <name>Task 2: Upgrade to Tailwind CSS 4.0</name>
  <files>
    website/package.json
    website/postcss.config.mjs
    website/tailwind.config.ts
    website/src/app/globals.css
  </files>
  <action>
    From website/ directory:

    1. Remove Tailwind v3 and install v4:
       ```bash
       cd website
       npm uninstall tailwindcss autoprefixer
       npm install tailwindcss @tailwindcss/postcss
       ```

    2. Update postcss.config.mjs (replace entire file):
       ```javascript
       /** @type {import('postcss-load-config').Config} */
       const config = {
         plugins: ["@tailwindcss/postcss"],
       };

       export default config;
       ```

    3. Delete tailwind.config.ts (not needed in v4):
       ```bash
       rm tailwind.config.ts
       ```

    4. Update src/app/globals.css (replace entire file):
       ```css
       @import "tailwindcss";

       /* Custom theme will be added in later plans */
       @theme {
         /* Placeholder for Heebo font variable */
         --font-sans: system-ui, sans-serif;
       }
       ```

    5. Install all dependencies:
       ```bash
       npm install
       ```

    CRITICAL: Use @import "tailwindcss" NOT @tailwind directives (Tailwind 4.0 syntax).
    CRITICAL: Delete tailwind.config.ts file - v4 uses CSS-first @theme blocks.
  </action>
  <verify>
    Run `cd website && npm run build` completes without errors.
    Check postcss.config.mjs contains "@tailwindcss/postcss".
    Check globals.css contains `@import "tailwindcss"`.
    Confirm tailwind.config.ts is deleted.
  </verify>
  <done>
    Tailwind 4.0 configured with CSS-first approach, build passes.
  </done>
