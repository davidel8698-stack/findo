<task type="auto">
  <name>Task 1: Fix GSAP animation - use fromTo for explicit opacity control</name>
  <files>website/components/sections/hero/ActivityFeed.tsx</files>
  <action>
The root cause: `gsap.from()` animates FROM specified values TO current CSS state. Cards have `opacity-0` class, so animation goes from opacity:0 to opacity:0 - cards stay invisible forever.

Fix: Replace `gsap.from()` with `gsap.fromTo()` to explicitly set BOTH start AND end opacity values.

In the useGSAP callback (around line 76-84), change:

```typescript
// BEFORE (broken)
tl.from(".activity-card", {
  y: 40,
  opacity: 0,
  scale: 0.9,
  stagger: {
    each: 0.3,
    from: "start",
  },
});
```

TO:

```typescript
// AFTER (fixed)
tl.fromTo(
  ".activity-card",
  {
    // FROM values (start state)
    y: 40,
    opacity: 0,
    scale: 0.9,
  },
  {
    // TO values (end state)
    y: 0,
    opacity: 1,
    scale: 1,
    stagger: {
      each: 0.3,
      from: "start",
    },
  }
);
```

Keep the `opacity-0` class on ActivityCard (line 110) - it prevents flash of unstyled content before GSAP initializes.

Keep `will-change-transform` class and the removeWillChange cleanup callback.
  </action>

1. Run `npm run dev` in website directory
2. Open http://localhost:3000 in browser
3. Activity cards should animate in with bouncy staggered entrance
4. Cards should be fully visible (opacity 1) after animation completes
5. No TypeScript errors: `npx tsc --noEmit`
  </verify>
  <done>Activity cards animate from invisible to visible with bouncy spring physics, then hold final state</done>
