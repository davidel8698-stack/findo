  <action>
1. Build production version:
   ```bash
   cd website && npm run build
   ```

2. Start production server:
   ```bash
   npm run start
   ```

3. Run Lighthouse audit via CLI (more consistent than DevTools):
   ```bash
   # Install Lighthouse CLI if needed
   npm install -g lighthouse

   # Run mobile audit
   lighthouse http://localhost:3000 \
     --emulated-form-factor=mobile \
     --throttling-method=simulate \
     --output=json \
     --output-path=./lighthouse-report.json
   ```

4. Or use Chrome DevTools:
   - Open http://localhost:3000 in Chrome
   - Open DevTools > Lighthouse tab
   - Select: Mobile, Performance, Best Practices, Accessibility, SEO
   - Run audit

5. Analyze results and fix issues:

   **If Performance < 95:**
   - Check LCP: If > 1.5s, investigate hero rendering
   - Check TBT: If high, look for long tasks in JavaScript
   - Check CLS: If > 0, find layout shifts

   **Common fixes:**
   - Image optimization: Ensure next/image is used for all images
   - Font loading: Verify Heebo has display:swap
   - JavaScript: Check for blocking scripts
   - CSS: Look for unused CSS

6. Re-run audit after fixes until score >= 95

Document any issues found and fixes applied.
