```css
/* ============================================
 * 4PX-GRID SECTION RHYTHM TOKENS - Phase 29
 * Linear Design System vertical rhythm
 * Desktop values (mobile uses 60% scale via utilities)
 * ============================================ */

/* Section padding - desktop values */
--spacing-section-hero: 7.5rem;      /* 120px */
--spacing-section-feature: 5rem;     /* 80px */
--spacing-section-cta: 4rem;         /* 64px */
--spacing-section-footer: 3rem;      /* 48px */

/* Section padding - mobile values (60% scale) */
--spacing-section-hero-mobile: 4.5rem;    /* 72px */
--spacing-section-feature-mobile: 3rem;   /* 48px */
--spacing-section-cta-mobile: 2.5rem;     /* 40px */

/* Grid system tokens */
--spacing-gutter: 1.5rem;            /* 24px column gap */
--spacing-edge: 1rem;                /* 16px mobile edge margins */

/* Container max-width */
--container-content: 75rem;          /* 1200px */
```

NOTE: The existing spacing scale (--spacing-0 through --spacing-96) already covers the 4px grid values. Tailwind's default scale uses 0.25rem increments which maps to 4px. For example:
- --spacing-1 = 0.25rem = 4px
- --spacing-2 = 0.5rem = 8px
- --spacing-3 = 0.75rem = 12px
- --spacing-4 = 1rem = 16px
- --spacing-6 = 1.5rem = 24px
- --spacing-8 = 2rem = 32px
- --spacing-12 = 3rem = 48px
- --spacing-16 = 4rem = 64px
- --spacing-24 = 6rem = 96px
- --spacing-32 = 8rem = 128px

So SPACE-01 is already satisfied. We only need to add the semantic section tokens.
